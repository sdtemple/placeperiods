Model Build and Specifications

 We built a convolutional neural network using the Keras API. We rely on Google's Word2Vec model to give words a vector representation.

Network topology:
1 3-dimensional convolutional layer with 16 filters that convolves one word at a time
1 fully-connected layer with 12 nodes and rectified linear unit activations
1 fully-connected layer with 1 node and sigmoid activation

Model Parameters:
Optimizer - rmsprop
Loss Function - binary_crossentropy
Dropout - No
Training Epochs - 10
Batch Size - 30

 We trained out neural network on 64 formal essays written by high school and college students. We used the Talapas supercomputer cluster to access GPU nodes for a grid search of parameters. We observed only marginal differences between parameter choices. We tested our period placer on student recalls of short passages and achieved promising results. Area under the Receiver Operating Characteristic curve scored close to 1 for all recalls. Precision, Recall, and F1 from the confusion matrix scored strongly as well.

 Notably, our CNN lacked complexity, but we achieved strong results. We suspect this success comes from the ability of Google's Word2Vec model to track differences between words in a 300-dimensional vector space. A full stop marks a change in ideas. This shift often occurs with a change in diction. Hence, we can pick up on this change in word choice with content-based word vectorizations and some convolutional filters.
